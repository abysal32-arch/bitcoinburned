# task-03 — Reproducible build: bundle → tool page in one command

## Goal
Right now the bitcoinjs bundle inside `tool/index.html` was hand-spliced once; any future edit to `src/burn.js` has no reliable path into the browser app. Create `npm run build:tool` that rebuilds and re-splices it deterministically.

## Preconditions
task-01/02 done: `tool/index.html` exists and works; `tasks/task-02/psbt-vector.txt` holds a known-good input→PSBT vector; esbuild is in devDependencies.

## Steps
1. In `tool/index.html`, wrap the existing bundle script in markers (the bundle is the first `<script>` after the footer; its body starts `"use strict";(()=>{var ns=Object.defineProperty…`):
   ```html
   <!-- BUNDLE:START — generated by scripts/build-tool.js; never edit by hand -->
   <script> …existing bundle… </script>
   <!-- BUNDLE:END -->
   ```
   The second, readable `<script>` (form wiring, starts `(function(){`) stays OUTSIDE the markers.
2. Write `scripts/build-tool.js` (plain Node, no new deps):
   - Run esbuild via its JS API (`require('esbuild').buildSync`) with the same options as package.json's `build:web`: entry `src/browser-entry.js`, bundle, platform browser, iife, minify, outfile `dist/bundle.js`.
   - Read `tool/index.html`, replace everything between the markers with a fresh `<script>` + bundle + `</script>`, write it back. Fail loudly (non-zero exit) if the markers aren't found exactly once each.
3. package.json: add `"build:tool": "node scripts/build-tool.js"`.
4. **Gates**:
   - Run `npm run build:tool` — completes, `tool/index.html` regenerated.
   - **Determinism/regression**: open the regenerated tool page, enter the exact inputs from `tasks/task-02/psbt-vector.txt`, Build → PSBT base64 **must be byte-identical** to the stored vector (PSBT building has no randomness; any difference means the bundle regressed — stop and investigate).
   - Run it twice — second run produces an identical file (`git diff` clean after re-run).
   - `node test/burn.test.js` green.
5. README: replace the manual "npx esbuild … then splice" instructions with `npm run build:tool`.
6. Commit: `task-03: reproducible bundle build via npm run build:tool`.

## Do NOT
- Change `src/burn.js` logic.
- Reformat or prettify the tool page beyond inserting the two marker comments.

## Exit criteria
One command rebuilds the tool page; regression vector matches; repeat runs are idempotent; README updated; tests green; commit + STATUS updated; Joe told: safe to `/clear`, next = task-04.
